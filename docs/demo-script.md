# Sixty-second demo

Primary case: protected live run with `F-AMBIGUOUS-COMMIT`.

| Time   | Visible proof                                                             |
| ------ | ------------------------------------------------------------------------- |
| 0–4s   | “Agents fail after the click. This runtime makes them prove the outcome.” |
| 4–10s  | Editable trip constraints, ambiguous-commit seed, protected mode          |
| 10–16s | Immutable contract, exact approval, no blind retry, contract hash         |
| 16–23s | Real screenshot actions across GoMail and Northstar                       |
| 23–30s | Exact NS451, aisle, nonstop, `$389` approval                              |
| 30–36s | Rendered confirmation click; server commits; UI receives 504              |
| 36–43s | `OUTCOME UNKNOWN`; commit retry blocked; Manage Trip and GoMail checked   |
| 43–49s | Booking verified, then exactly one DayPlan update                         |
| 49–54s | One booking, one calendar update, zero unauthorized or duplicate effects  |
| 54–57s | Clearly labeled recorded replay                                           |
| 57–60s | Real paired result, raw rows, source link                                 |

Price drift and safe abort are selectable secondary demonstrations.
